# 网络访问部署完整指南

让你的项目可以在互联网上被任何人访问！

---

## 🌟 方案一：GitHub Pages（免费、推荐）

这是最简单、免费的方式！

### 步骤 1：推送到 GitHub

```bash
# 初始化 Git
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 创建 GitHub 仓库（去 https://github.com/new 创建）
# 仓库名：todoList

# 关联并推送
git remote add origin https://github.com/qingchuan73/todoList.git
git branch -M main
git push -u origin main
```

### 步骤 2：启用 GitHub Pages

1. 打开你的 GitHub 仓库：https://github.com/qingchuan73/todoList
2. 点击右上角的 **Settings**
3. 左侧菜单找到 **Pages**（在 "Code and automation" 下面）
4. 在 **Build and deployment** 部分：
   - **Source**: 选择 `GitHub Actions`
   - 点击 **Save**

### 步骤 3：等待自动部署

- 推送代码后，GitHub Actions 会自动运行
- 点击仓库顶部的 **Actions** 标签查看进度
- 大约 2-5 分钟后部署完成

### 步骤 4：访问你的网站

部署成功后，访问：

```
https://qingchuan73.github.io/todoList/
```

把这个链接发给任何人，他们都能看到！

## 🌟 方案二：Vercel（免费、更简单）

Vercel 部署更快，支持自动 HTTPS！

### 步骤 1：注册 Vercel

访问 https://vercel.com，用 GitHub 账号登录

### 步骤 2：导入项目

1. 点击 **Add New...** → **Project**
2. 选择你的 `todoList` 仓库
3. 点击 **Import**

### 步骤 3：配置部署

在配置页面：
- **Framework Preset**: 选择 `Other`
- **Root Directory**: 保持默认
- **Build Command**: 留空（或者填 `npm run build:all`）
- **Output Directory**: 留空

点击 **Deploy**

### 步骤 4：完成！

1-2 分钟后就部署好了！你会得到一个类似这样的链接：

```
https://todolist-abc123.vercel.app
```

可以在 Vercel 控制台自定义域名！

## 🌟 方案三：Netlify（免费、拖拽部署）

### 方法 A：Git 自动部署

1. 访问 https://netlify.com，注册登录
2. 点击 **Add new site** → **Import an existing project**
3. 选择 GitHub，选择你的仓库
4. 点击 **Deploy site**

### 方法 B：拖拽部署（最简单）

1. 先在本地构建所有项目：
```bash
npm run install:all
npm run build:all
```

2. 访问 https://app.netlify.com/drop
3. 把整个项目文件夹拖拽进去
4. 完成！得到一个链接

## ❓ 常见问题

### Q: 部署后页面是空白的？
A: 检查各框架的构建输出目录是否正确，确保 dist 文件夹已生成

### Q: 链接可以改成我自己的域名吗？
A: 可以！GitHub Pages、Vercel、Netlify 都支持绑定自定义域名

### Q: 每次更新代码都要重新部署吗？
A: 不用！配置好 Git 集成后，每次 push 代码会自动部署

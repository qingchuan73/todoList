# 简历项目描述

## 项目名称
TodoList - 多框架对比项目

## 项目简介
基于 TodoMVC 规范，使用 React 19、Vue 3、Angular 20、Vanilla JS 四种主流技术栈实现的待办事项应用对比展示项目。

## 技术栈
- **前端框架**：React 19.1.1、Vue 3.5.21、Angular 20.3.0
- **构建工具**：Vite 7、Angular CLI、Webpack
- **开发语言**：TypeScript、JavaScript ES6+
- **代码规范**：Prettier、ESLint

## 项目职责
1. **架构设计与实现**
   - 设计统一的 TodoMVC 功能规范
   - 使用四种技术栈独立实现完整功能
   - 对比各框架的开发模式和性能差异

2. **工程化建设**
   - 配置 Vite 实现快速开发和构建优化
   - 统一代码规范，集成 Prettier 格式化工具
   - 设计 monorepo 结构管理多框架代码

3. **功能开发**
   - 实现任务增删改查、状态切换、筛选功能
   - 本地存储持久化数据
   - 响应式布局和交互动画

## 项目亮点
- 🚀 **技术广度**：同时掌握 React、Vue、Angular 三大主流框架
- 💡 **学习能力**：使用各框架最新版本（React 19、Angular 20）
- 📦 **工程化**：配置 Vite 构建工具，实现开发环境热更新
- 🎯 **代码质量**：TypeScript 类型安全，统一代码规范

## 面试官可能关注的问题

1. **为什么选择这四个框架进行对比？**
   - 展示了对前端生态的全面了解
   - 涵盖了从企业级到轻量级的各种选择

2. **各框架的优缺点对比？**
   - React：生态丰富，灵活性高，但需要更多选型决策
   - Vue：上手简单，文档完善，国内生态好
   - Angular：企业级框架，功能完整，但学习曲线陡峭
   - Vanilla JS：无依赖，性能最佳，但开发效率低

3. **项目中遇到的难点是什么？**
   - 可以谈统一功能规范在不同框架的实现差异
   - 或者状态管理在各框架的不同处理方式

4. **如果让你推荐一个框架，你会怎么选？**
   - 根据团队情况、项目规模、维护周期等因素综合考虑
